教材制作・修正 お問い合わせフォーム - デプロイ手順
====================================================

このフォルダには3つのファイルがあります。
・index.html … 依頼者が入力する画面（プレビューはブラウザで直接開いても見られます）
・Code.gs     … 送信されたらメールを送るバックエンド処理
・README.txt  … このファイル

【重要】学校向け教材のGAS連携版（台形の面積などのGAS連携版）とは
「アクセスできるユーザー」の設定が違います。こちらは誰でも使える
公開フォームなので、下の【3】で必ず「全員」を選んでください。


【1】Google Apps Script プロジェクトを作る
  1. Google ドライブを開く（kyozai.cafe@gmail.com でログイン推奨）
  2. 「新規」→「その他」→「Google Apps Script」
     （見当たらない場合は https://script.google.com/ から「新しいプロジェクト」）
  3. プロジェクト名を「教材お問い合わせフォーム」などにする

【2】コードを貼り付ける
  1. 左側のファイル一覧にある「Code.gs」を開き、中身をすべて削除
  2. 渡した Code.gs の内容をすべて貼り付けて保存（Ctrl+S）
     ・冒頭の CONTACT_TO の値が送信先メールアドレスです。
       kyozai.cafe@gmail.com 以外に送りたい場合はここを書き換えてください。
  3. 左側の「＋」→「HTML」を選び、ファイル名を必ず「index」にする（拡張子は自動でつきます）
  4. 渡した index.html の内容をすべて貼り付けて保存

【3】ウェブアプリとして公開する
  1. 右上の「デプロイ」→「新しいデプロイ」
  2. 種類の選択（歯車マーク）で「ウェブアプリ」を選ぶ
  3. 「次のユーザーとして実行」→ 自分（kyozai.cafe@gmail.com）
  4. 「アクセスできるユーザー」→ 必ず「全員」を選ぶ
     （学校の教材とちがい、これはサイトの来訪者なら誰でも送れる
     お問い合わせフォームなので、ドメイン限定にしないでください）
  5. 「デプロイ」をクリックし、表示されたウェブアプリのURL（.../exec で終わるURL）をコピーする
  6. 初回のみ、権限の承認画面が出るので許可する
     （「Googleで確認されていないアプリです」と出ますが、自分で作った
     アプリなので「詳細」→「（プロジェクト名）に移動（安全ではないページ）」で進めて大丈夫です）

【4】動作確認
  1. コピーしたURLをブラウザで開く
  2. 名前・連絡先などを適当に入力して「依頼文を生成する」
  3. 「📤 この内容を送信する」を押す
  4. kyozai.cafe@gmail.com（または書き換えた宛先）にメールが届けば成功です

【5】サイトに載せる
  ・動作確認ができたら、この .../exec のURLを教えてください。
    kyozai-cafe の /privacy ページの「お問い合わせ」から、このURLに
    リンクするよう更新します。

【困ったときは】
  ・コードを直接書き換えても大丈夫です。次に相談するときは
    「Code.gsの◯◯を直したい」のように伝えてもらえれば調整できます。
  ・送信先を増やしたい場合は、Code.gs の MailApp.sendEmail(...) を
    複数回呼ぶか、宛先をカンマ区切りにすることでも可能です（相談してください）。
